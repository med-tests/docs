#### Общая информация о базе
три файла, связанных с базой данных SQLite:
1. **`data.db`** – основной файл базы данных. Содержит все таблицы, индексы и данные в структурированном виде. Именно в нём хранится постоянная информация.
2. **`data.db-wal`** – файл **Write-Ahead Log** (журнал упреждающей записи). В режиме WAL (включён через `db.pragma('journal_mode = WAL')`) все изменения сначала записываются в этот файл, а затем асинхронно переносятся в основной `data.db`. Это повышает производительность при параллельных операциях чтения/записи и снижает блокировки.
3. **`data.db-shm`** – файл **shared memory** (разделяемая память). Используется для координации доступа к WAL-файлу между несколькими процессами/потоками, работающими с одной базой данных. Хранит внутреннюю служебную информацию (например, указатели на последние записи).

При нормальной работе все три файла должны присутствовать одновременно. Если вы закроете соединение с БД (вызов `db.close()`), SQLite не удаляет автоматически `-wal` и `-shm` – они остаются и используются при следующем открытии.

Периодически SQLite выполняет **контрольную точку** (checkpoint), когда данные из `-wal` переносятся в основной файл, после чего `-wal` может уменьшиться в размере или стать пустым, но сам файл обычно не удаляется.

Режим WAL является рекомендуемым для большинства приложений, особенно когда чтение и запись выполняются конкурентно, поэтому появление этих трёх файлов – штатное поведение.

#### Сделать бэкап и скачать его на локальный комп
1. Зайти на продакшн-сервер
2. Перейти в `home\tanya\back`
3. Запустить команду `npm run backup`.  Она сделает бэкап базы в корне проекта.
4. Посмотреть и скопировать имя файла. `ls -l`
5. Выйти с сервера
6. Перейти в папку с бэкапами, `med-tests/backups`, запустить из нее WSL
7. В WSL запустить командy 
   `scp tanya@chesnok:/home/tanya/back/backup_06-07-26_05-21.db ./`
   (`scp username@server_ip:/путь/к/backup_файлу.db /путь/на/локальной/машине/`)
8. Проверить, что в целевой папке появился бэкап



#### Загрузить бэкап с локального компа и заменить им базу
1. Загрузить бэкап. Зайти в папку с бэкапами `med-tests/backups`. Скопировать название нужного бэкапа. Запустить из нее WSL. 
   В WSL `scp <название бэкапа>.db tanya@chesnok:/home/tanya/back`
2. Зайти на продакшн-сервер
3. Перейти в `home\tanya\back`
4. Зайти в скрин (`screen -r`), остановить приложение 
5. Переименоват
6. ь старые файлы c префиксом `old-`
   `mv data.db old-data.db`
   `mv data.db-wal old-data.db-wal`
   `mv data.db-shm old-data.db-shm`
6. Переименовать бэкап в `data.db` , например
   `mv backup_2026-07-06.db data.db
7. Зайти в скрин `screen -r` и  перезапустить проект `npm start`
8. Проверить, что все благополучно
9. Удалить старые файлы `rm old-data.db old-data.db-wal old-data.db-shm`
