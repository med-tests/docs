В рамках проекта тэги указывают на коммит, с которым работает продакшн.
Используются аннотированные тэги, чтобы иметь возможность посмотреть как часто происходят изменения и какого рода.

1. `git desribe --tags`  - посмотреть последний тэг и сколько после него коммитов
2. Придумать название тэга по SemVer (мажор.минор.патч)
3. `git tag -a <название тэга> -m "<описание>"`
4. `sudo git push --tags` - тэги сами себя не запушат

Иногда тэги еще и сами себя не запулят `git pull --tags` или `git fetch --tags`


Чтобы посмотреть историю (самый свежий тэг сверху)
``` bash
git tag --sort=-creatordate --format='%(refname:short) - %(creatordate:format:%d.%m.%Y) - %(subject)'

```
