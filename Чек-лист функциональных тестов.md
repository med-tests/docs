[[Med-tests]]

##### Логин (/login)
###### Дефолтное
- [ ] если у юзера нет jwt-токена, бэк возвращает ошибку и происходит редирект на экран логина
- [ ] если jwt-токен юзера устарел (24 часа), бэк возвращает ошибку и происходит редирект на экран логина
- [ ] если jwt-токен юзера устарел (24 часа), он должен быть удален из локального хранилища
###### Регистрация
- [ ] есть форма регистрации
- [ ] в форме регистрации есть текстовое обязательное поле `Логин`
- [ ] в форме регистрации есть текстовое обязательное поле `Пароль`
- [ ] у пароля есть переключатель - пароль отображен или скрыт за звездочками
- [ ] в форме регистрации есть переключение на форму входа
- [ ] в форме регистрации есть кнопка `Зарегистрироваться`
- [ ] перед отправкой запроса на регистрацию происходит валидация полей
- [ ] логин должен быть от 2 до 16 символов длиной
- [ ] пароль должен быть от 8 символов длиной
- [ ] пароль должен содержать как минимум 1 цифру
- [ ] пароль должен содержать как минимум 1 заглавную латинскую букву
- [ ] пароль должен содержать как минимум 1 строчную латинскую букву
- [ ] пароль должен содержать как минимум 1 спец. символ
- [ ] если логин уже используется, бэк вернет ошибку
- [ ] если регистрация успешна, jwt-токен будет сохранен в локальное хранилище
- [ ] если регистрация успешна, произойдет переход на экран с графиками

###### Вход
- [ ] есть форма входа
- [ ] в форме входа есть текстовое обязательное поле `Логин`
- [ ] в форме входа есть текстовое обязательное поле `Пароль`
- [ ] у пароля есть переключатель - пароль отображен или скрыт за звездочками
- [ ] в форме входа есть переключение на форму регистрации
- [ ] в форме входа есть кнопка `Войти`
- [ ] если логин или пароль введены неверно, бэк вернет ошибку
- [ ] если вход успешен, jwt-токен будет сохранен в локальное хранилище
- [ ] если вход успешен, произойдет переход на экран с графиками

###### Выход
 - [ ] jwt-токен будет удален из локального хранилища
 - [ ] переход на экран логина


##### Отображение анализа
- [ ] юзер получает только свои анализы
- [ ] юзер получает только те анализы, которые не были удалены
- [ ] если нет ни одного анализа, должно быть выведено сообщение об этом
- [ ] если все графики скрыты, должно быть выведено сообщение об этом
###### В списке анализов
- [ ] анализ есть в списке анализов
- [ ] у анализа есть название
- [ ] у анализа есть кнопки редактирования, видимости и удаления (не уверена, что это должно быть тут)
- [ ] при наведении на кнопку редактировать появляется всплывающая подсказка (не уверена, что это должно быть тут)
- [ ] при наведении на кнопку видимости появляется всплывающая подсказка (не уверена, что это должно быть тут)
- [ ] при наведении на удаления видимости появляется всплывающая подсказка (не уверена, что это должно быть тут)
- [ ] при клике на название анализа осуществляется скрол к анализу в списке графиков, если их больше одного

###### В списке графиков
- [ ] анализ есть в списке графиков
- [ ] у анализа выведено название
- [ ] у анализа выведены границы нормы (если указаны)
- [ ] при наведении на границы нормы (если указаны), появляется всплывающая подсказка
- [ ] у анализа есть поля для фильтрации результатов по дате - начало периода и конец периода
- [ ] если у анализа нет результатов - выведено сообщение об этом и нет графика
- [ ] если у анализа нет результатов - начало периода и конец периода пустые
- [ ] если у анализа есть один результат - выведен график
- [ ] если у анализа есть один результат - начало и конец периода содержат дату сдачи анализа
- [ ] если у анализа есть несколько результатов - выведен график
- [ ] если у анализа есть несколько результатов - начало периода содержит дату самого раннего анализа (после создания или сброса)
- [ ] если у анализа есть несколько результатов - конец периода содержит дату самого позднего анализа (после создания или сброса)
- [ ] если у анализа есть несколько результатов - в календарях выбора периодов выделены даты сдачи анализов
- [ ] если у анализа есть несколько результатов - нельзя выбрать дату начала периода позже, чем дата конца периода
- [ ] если у анализа есть несколько результатов - нельзя выбрать дату конца периода раньше, чем дата начала периода
- [ ] если у анализа указаны границы нормы - они выводятся в формате `✅ от <число> до <число>`
- [ ] если у анализа указана только нижняя граница нормы - она выводится в формате `✅ от <число>`
- [ ] если у анализа указана только верхняя граница нормы - она выводится в формате `✅ до <число>`
- [ ] если у анализа не указаны границы нормы, ничего не выводится
- [ ] при наведении на иконку в выводе границы норм должна появляться всплывающая подсказка

###### Изменение границ выводимых на графике значений
- [ ] при клике на поле ввода периода появляется календарь
- [ ] поле ввода заблокировано для ручного ввода
- [ ] при клике на ту же самую дату ничего не происходит
- [ ] даты сдачи анализов выделены в календаре вне зависимости от выбранного периода
- [ ] выбранная дата выделена в календаре
- [ ] нельзя выбрать дату раньше даты самого первого анализа
- [ ] нельзя выбрать дату позже самого последнего анализа
- [ ] `Начало периода` - при клике на кнопку очистить устанавливается дата самого первого анализа
- [ ] `Конец периода` - при клике на кнопку очистить устанавливается дата самого последнего анализа
- [ ] При изменении дат происходит перерисовка графика - остаются только результаты анализов, сданных в период между `Началом периода` и `Концом периода` включая эти даты
- [ ] при перезагрузке страницы - выбранные даты сохраняются
- [ ] при перезагрузке страницы - график результатов соответствует выбранным датам
- [ ] изменение дат влияет только на свой график
- [ ] при отсутствии результатов анализов в выбранный период - график скрывается
- [ ] при отсутствии результатов анализов в выбранный период - выводится сообщение об отсутствии результатов за период
- [ ] если у анализа нет результатов - нельзя выбрать даты начала и конца периода 


##### Добавление анализа
- [ ] есть кнопка добавление анализа
- [ ] при нажатии на кнопку открывается модалка
- [ ] в модалке есть поля ввода для названия, верхней границы, нижней границы, для результатов и и их дат
- [ ] у модалки есть кнопки Добавить анализ и Отменить
- [ ] поле название обязательное
- [ ] название не длиннее 45 символов
- [ ] поля результат и дата обязательные
- [ ] верхняя граница должна быть больше или равна, чем нижняя граница
- [ ] можно создать анализ без результатов
- [ ] можно создать анализ с 1 результатом
- [ ] можно создать анализ с несколькими результатами
- [ ] после создания анализ добавлен в список анализов и списке графиков
- [ ] после создания анализ в списке графиков должен соответствовать [[Чек-лист функциональных тестов#В списке графиков|Чек--лист анализа в списке графиков]]
- [ ] после создания анализа в списке анализов должен соответствовать [[Чек-лист функциональных тестов#В списке анализов|Чек--лист анализа в списке анализов]]
- [ ] после создания анализ должен быть видим (кнопка изменения видимости)
- [ ] при закрытии модалки(после отмены или отправки) и повтором открытии модалки, поля будут очищены

##### Добавление результатов анализа
- [ ] новому анализу можно добавить результаты
- [ ] существующему анализу можно добавить результаты

##### Удаление анализа
- [ ] при нажатии на кнопку удаления анализа в списке анализов появляется модалка подтверждения удаления
- [ ] у модалки есть кнопки Удалить и Отменить
- [ ] тест предупреждения содержит название анализа
- [ ] при нажатии на кнопку Удалить анализа скрывается из списка анализов и списка графиков
- [ ] при перезагрузке старицы удаленный анализ не появляется в списке анализов и в списке графиков

##### Удаление результатов анализа
- [ ] у нового анализа можно удалить результаты
- [ ] у существующего анализа можно удалить результаты

##### Редактирование анализа
  ###### Изменение названия
 - [ ] при нажатии на кнопку редактирования открывается модалка редактирования
 - [ ] в форме есть поле `Название`
 - [ ] поле содержит текущее название анализа 
 - [ ] нельзя сделать название длиннее 45 символов
 - [ ] при закрытии модалки (после отмены или отправки) и повтором открытии для другого анализа, название должно измениться соответствующе
 - [ ] при внесении изменений без сохранения и повтором открытии модалки поле должно содержать актуальное название анализа
 - [ ] нельзя сохранить анализ без названия
 - [ ] после изменения названия и сохранения, название должно поменяться в списке анализов и в списке графиков

  
  ###### Изменение границ нормальны значений 
  - [ ] при нажатии на кнопку редактирования открывается модалка редактирования
  - [ ] в форме есть поле `Верхняя граница нормы` / `Нижняя граница нормы`
  - [ ] поле содержит текущую указанную границу нормы или пустую строку
  - [ ] при закрытии модалки (после отмены или отправки) и повтором открытии для другого анализа, значение должно измениться соответствующе вновь открытому анализу
 - [ ] при внесении изменений без сохранения и повтором открытии модалки поле должно содержать сохраненное значение или пустую строку. 
 - [ ] можно сохранить анализ без верхней границы нормы
 - [ ] можно сохранить анализ без нижней границы нормы
 - [ ] можно сохранить анализ без указания норм
 - [ ] можно сохранить анализ только с верхней границей нормы
 - [ ] можно сохранить анализ только с нижней границей нормы
 - [ ] можно сохранить анализ с обоими границами
 - [ ] нельзя сохранить анализ, если нижняя граница больше верхней
 - [ ] после изменения названия и сохранения - должен поменяться текст нормы в списке графиков. 
       Если указаны обе границы - `✅ от <число> до <число>`
       Если указана только нижняя граница - `✅ от <число>`
       Если указана только верхняя граница -`✅ до <число>`
       Если границы не указаны - текст скрыт
 - [ ] после изменения названия и сохранения - должен произойти перерасчет цветов точек в графике (синий - ниже нормы, зеленый - в пределах нормы, красный - выше нормы)
  
  ###### Изменение видимости анализа
  **Если анализ видим**
- [ ] иконка в кнопке изменения видимости не зачеркнута
- [ ] название анализа ярким цветом
- [ ] анализ виден в списке графиков 
- [ ] при нажатии на кнопку изменения видимости, анализ должен быть скрыт из списка графиков
- [ ] при нажатии на кнопку изменения видимости, иконка кнопки должна измениться на перечеркнутую
- [ ] при нажатии на кнопку изменения видимости, цвет названия должен стать бледнее
- [ ] после перезагрузки страницы анализ должен быть скрыт в списке графиков
- [ ] после перезагрузки страницы иконка кнопки изменения видимости должна быть зачеркнута
- [ ] после перезагрузки страницы название анализа должно быть бледного цвета

**Если анализ скрыт**
- [ ] иконка в кнопке изменения видимости зачеркнута
- [ ] название анализа бледным цветом
- [ ] анализ не показан в списке графиков
- [ ] при нажатии на кнопку изменения видимости, анализ должен быть показан из списка графиков
- [ ] при нажатии на кнопку изменения видимости, иконка кнопки должна измениться на не перечеркнутую
- [ ]  при нажатии на кнопку изменения видимости, цвет названия должен стать ярче
- [ ] после перезагрузки страницы анализ должен быть показан в списке графиков
- [ ] после перезагрузки страницы иконка кнопки изменения видимости должна быть не зачеркнута
- [ ] после перезагрузки страницы название анализа должно быть яркого цвета
  
###### Изменение результатов анализа
  - [ ] у существующего анализа можно изменить даты результатов
  - [ ] у существующего анализа можно изменить значение результатов
  - [ ] нельзя сохранить результат без даты
  - [ ] нельзя сохранить результат без значения
  - [ ] можно удалить все результаты и сохранить. График должен быть скрыт и выведено сообщение.
