[[Med-tests]]

##### Логин (/login)
###### Дефолтное
- [ ] если у юзера нет jwt-токена, бэк возвращает ошибку и происходит редирект на экран гостевую страницу
- [ ] если jwt-токен юзера устарел (24 часа), бэк возвращает ошибку и происходит редирект на экран логина
- [ ] если jwt-токен юзера устарел (24 часа), он должен быть удален из локального хранилища
###### Регистрация
- [ ] к регистрации можно перейти из формы логина (/login)
- [ ] есть форма регистрации
- [ ] в форме регистрации есть текстовое обязательное поле `Логин`
- [ ] в форме регистрации есть текстовое обязательное поле `Пароль`
- [ ] у пароля есть переключатель - пароль отображен или скрыт за звездочками
- [ ] в форме регистрации есть переключение на форму входа
- [ ] в форме регистрации есть кнопка `Зарегистрироваться`
- [ ] перед отправкой запроса на регистрацию происходит валидация полей
- [ ] логин должен быть от 2 до 16 символов длиной
- [ ] пароль должен быть от 8 символов длиной
- [ ] пароль должен содержать как минимум 1 цифру
- [ ] пароль должен содержать как минимум 1 заглавную латинскую букву
- [ ] пароль должен содержать как минимум 1 строчную латинскую букву
- [ ] пароль должен содержать как минимум 1 спец. символ
- [ ] если логин уже используется, бэк вернет ошибку
- [ ] если регистрация успешна, jwt-токен будет сохранен в локальное хранилище
- [ ] если регистрация успешна, произойдет переход на экран с графиками

###### Вход
- [ ] есть форма входа
- [ ] в форме входа есть текстовое обязательное поле `Логин`
- [ ] в форме входа есть текстовое обязательное поле `Пароль`
- [ ] у пароля есть переключатель - пароль отображен или скрыт за звездочками
- [ ] в форме входа есть переключение на форму регистрации
- [ ] в форме входа есть кнопка `Войти`
- [ ] если логин или пароль введены неверно, бэк вернет ошибку
- [ ] если вход успешен, jwt-токен будет сохранен в локальное хранилище
- [ ] если вход успешен, произойдет переход на экран с графиками

###### Выход
 - [ ] jwt-токен будет удален из локального хранилища
 - [ ] переход на экран логина


##### Отображение графиков
- [ ] юзер получает только свои данные
- [ ] юзер получает только те данные (показатели и их результаты), которые не были им удалены
- [ ] если нет ни одного показателя, должно быть выведено сообщение об этом
- [ ] если все графики скрыты, должно быть выведено сообщение об этом
###### В панели управления
- [ ] график есть в списке графиков
- [ ] у графика есть название
- [ ] у графика есть кнопки редактирования, видимости и удаления
- [ ] при наведении на кнопку редактировать появляется всплывающая подсказка
- [ ] при наведении на кнопку видимости появляется всплывающая подсказка
- [ ] при наведении на удаления видимости появляется всплывающая подсказка
- [ ] при клике на название анализа осуществляется скрол к анализу в списке графиков, если их больше одного

###### В списке графиков
- [ ] график показатель отображается
- [ ] у графика выведено название
- [ ] у графика выведены границы нормы (если указаны)
- [ ] если у графика есть результаты в выбранном периоде - выведено их среднее значение
- [ ] при наведении на границы нормы (если указаны), появляется всплывающая подсказка
- [ ] у графика есть поля для фильтрации результатов по дате - начало периода и конец периода
- [ ] если у графика нет результатов - выведено сообщение об этом и нет графика
- [ ] если у графика нет результатов - начало периода и конец периода пустые
- [ ] если у графика нет результатов - начало периода и конец периода невозможно изменить
- [ ] если у графика есть один результат - выведен график
- [ ] если у графика есть один результат - начало и конец периода содержат дату сдачи анализа
- [ ] если у графика есть один результат - начало и конец периода можно изменить
- [ ] если у анализа есть несколько результатов - выведен график
- [ ] если у анализа есть несколько результатов - начало периода содержит дату самого раннего анализа (после создания или сброса)
- [ ] если у анализа есть несколько результатов - конец периода содержит дату самого позднего анализа (после создания или сброса)
- [ ] если у анализа есть несколько результатов - в календарях выбора периодов выделены даты сдачи анализов
- [ ] если у анализа есть несколько результатов - нельзя выбрать дату начала периода позже, чем дата конца периода
- [ ] если у анализа есть несколько результатов - нельзя выбрать дату конца периода раньше, чем дата начала периода
- [ ] если у анализа указаны границы нормы - они выводятся в формате `✅ от <число> до <число>`
- [ ] если у анализа указана только нижняя граница нормы - она выводится в формате `✅ от <число>`
- [ ] если у анализа указана только верхняя граница нормы - она выводится в формате `✅ до <число>`
- [ ] если у анализа не указаны границы нормы, ничего не выводится
- [ ] при наведении на иконку в выводе границы норм должна появляться всплывающая подсказка

###### Изменение границ выводимых на графике значений
- [ ] при клике на поле ввода периода появляется календарь
- [ ] поле ввода заблокировано для ручного ввода
- [ ] при клике на ту же самую дату ничего не происходит
- [ ] даты сдачи анализов (результатов) выделены в календаре вне зависимости от выбранного периода
- [ ] выбранная дата выделена в календаре
- [ ] нельзя выбрать дату раньше даты самого раннего результата анализа
- [ ] нельзя выбрать дату позже самого последнего результата анализа
- [ ] `Начало периода` - при клике на кнопку очистить устанавливается дата самого раннего результата анализа
- [ ] `Конец периода` - при клике на кнопку очистить устанавливается дата самого последнего результата анализа
- [ ] При изменении дат происходит перерисовка графика - остаются только результаты анализов, сданных в период между `Началом периода` и `Концом периода` включая эти даты
- [ ] при перезагрузке страницы - выбранные даты сохраняются
- [ ] при перезагрузке страницы - график результатов соответствует выбранным датам
- [ ] изменение дат влияет только на свой график
- [ ] при отсутствии результатов анализов в выбранный период - график скрывается
- [ ] при отсутствии результатов анализов в выбранный период - выводится сообщение об отсутствии результатов за период
- [ ] если у анализа нет результатов - нельзя выбрать даты начала и конца периода 
##### Создания показателя
- [ ] есть кнопка создания показателя в контекстном меню панели управления
- [ ] при нажатии на кнопку открывается модалка
- [ ] в модалке есть поля ввода для названия, верхней границы, нижней границы, для результатов и и их дат
- [ ] у модалки есть кнопки Сохранить и Отменить
- [ ] поле название обязательное
- [ ] название не длиннее 45 символов
- [ ] поля результат и дата обязательные
- [ ] верхняя граница должна быть больше или равна, чем нижняя граница
- [ ] можно создать показатель без результатов
- [ ] можно создать показатель с 1 результатом
- [ ] можно создать показатель с несколькими результатами
- [ ] при закрытии модалки (после отмены или отправки) и повтором открытии модалки, поля должны быть очищены
- [ ] после создания показатель добавлен в список в панели управления и в список графиков
- [ ] после создания показатель в списке графиков должен соответствовать [[Чек-лист функциональных тестов#В панели управления|Чек-лист показателя в панели управления]]
- [ ] после создания показатель в списке анализов должен соответствовать [[Чек-лист функциональных тестов#В списке графиков|Чек-лист анализа в списке графиков]]
- [ ] после создания показатель в панели управления должен быть видим (кнопка изменения видимости)

##### Добавление результатов
- [ ] новому показателю можно добавить результаты
- [ ] существующему показателю можно добавить результаты
- [ ] у результата должны быть заполнены дата и значение
- [ ] значение должно быть числом
- [ ] дата должна быть в формате `DD.MM.YYYY`
- [ ] дата не должна быть в будущем

##### Удаление показателя
- [ ] при нажатии на кнопку удаления показателя в списке в панели управления появляется модалка подтверждения удаления
- [ ] у модалки есть кнопки Удалить и Отменить
- [ ] текст предупреждения содержит название показателя
- [ ] при нажатии на кнопку Удалить, показатель скрывается из списка в панели управления и в списка графиков
- [ ] при перезагрузке старицы удаленный анализ не появляется в списке в панели управления и в списке графиков

##### Редактирование показателя
  ###### Изменение названия
 - [ ] при нажатии на кнопку редактирования открывается модалка редактирования
 - [ ] в форме есть поле `Название`
 - [ ] поле содержит текущее название показателя 
 - [ ] нельзя сделать название длиннее 45 символов
 - [ ] при закрытии модалки (после отмены или отправки) и повтором открытии для другого анализа, название должно измениться соответствующе
 - [ ] при внесении изменений без сохранения и повтором открытии модалки поле должно содержать изначальное название показателя
 - [ ] нельзя сохранить показатель без названия
 - [ ] после изменения названия и сохранения, название должно поменяться в панели управления и в списке графиков

  
  ###### Изменение границ нормальны значений 
  - [ ] при нажатии на кнопку редактирования открывается модалка редактирования
  - [ ] в форме есть поля `Верхняя граница нормы` / `Нижняя граница нормы`
  - [ ] поля содержат текущую указанную границу нормы (положительное число или 0) или пустую строку
  - [ ] при закрытии модалки (после отмены или отправки) и повтором открытии для другого анализа, значение должно измениться на соответствующе вновь открытому
 - [ ] при внесении изменений без сохранения и повтором открытии модалки поле должно содержать сохраненное (предыдущее) значение или пустую строку
 - [ ] можно сохранить показатель без указания норм
 - [ ] можно сохранить показатель только с верхней границей нормы
 - [ ] можно сохранить показатель только с нижней границей нормы
 - [ ] можно сохранить показатель с обоими границами
 - [ ] нельзя сохранить показатель, если нижняя граница больше верхней
 - [ ] после изменения названия и сохранения - должен поменяться текст нормы в списке графиков. 
       Если указаны обе границы - `✅ от <число> до <число>`
       Если указана только нижняя граница - `✅ от <число>`
       Если указана только верхняя граница -`✅ до <число>`
       Если границы не указаны - текст скрыт
 - [ ] после изменения названия и сохранения - должен произойти перерасчет цветов точек в графике (синий - ниже нормы, зеленый - в пределах нормы, красный - выше нормы)
  
  ###### Изменение видимости показателя
  **Если показатель видим**
- [ ] иконка в кнопке изменения видимости не зачеркнута
- [ ] название показатель ярким цветом
- [ ] показатель отображен в списке графиков 
- [ ] при нажатии на кнопку изменения видимости, показатель должен быть скрыт из списка графиков
- [ ] при нажатии на кнопку изменения видимости, иконка кнопки должна измениться на перечеркнутую
- [ ] при нажатии на кнопку изменения видимости, цвет названия должен стать бледнее
- [ ] после перезагрузки страницы показатель должен быть скрыт в списке графиков
- [ ] после перезагрузки страницы иконка кнопки изменения видимости должна быть зачеркнута
- [ ] после перезагрузки страницы название показателя должно быть бледного цвета

  **Если показатель скрыт**
- [ ] иконка в кнопке изменения видимости зачеркнута
- [ ] название показателя бледным цветом
- [ ] показатель не показан в списке графиков
- [ ] при нажатии на кнопку изменения видимости, показатель должен быть показан в списке графиков
- [ ] при нажатии на кнопку изменения видимости, иконка кнопки должна измениться на не перечеркнутую
- [ ]  при нажатии на кнопку изменения видимости, цвет названия должен стать ярче
- [ ] после перезагрузки страницы показатель должен быть показан в списке графиков
- [ ] после перезагрузки страницы иконка кнопки изменения видимости должна быть не зачеркнута
- [ ] после перезагрузки страницы название показатель должно быть яркого цвета
  
###### Изменение результатов анализа
  - [ ] у существующего показатель можно изменить даты результатов
  - [ ] у существующего показатель можно изменить значение результатов
  - [ ] нельзя сохранить результат без даты
  - [ ] нельзя сохранить результат без значения
  - [ ] можно удалить все результаты и сохранить. График должен быть скрыт и выведено соответствующие сообщение

##### Удаление результатов
  - [ ] у нового показателя можно удалить результаты
  - [ ] у существующего показателя можно удалить результаты
